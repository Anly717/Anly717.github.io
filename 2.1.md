# PRD Lite（简版需求说明）

请在第 2 课完成本表。

---

## 一、项目名称

项目名称：大肥羊校园闲置物品交换平台（小程序 / 网页）

## 二、背景与目标

### 1. 项目背景
为什么会想到做这个项目？

答：同学处理闲置物品通常只能发朋友圈、群聊或校园墙，信息容易被刷屏淹没；买家难检索、分类不清、沟通成本高，导致交易效率低、闲置浪费。

### 2. 项目目标
我们这次最想做成什么？

答：做出一个“校内可用”的闲置发布与检索首页：同学能在首页完成搜索/分类浏览，看到最新与热门物品，并能快速发布闲置、联系卖家，形成校内流转闭环（不做支付/跨校/物流）。

## 三、目标用户

### 1. 目标用户是谁？
答：本校在校大学生（有闲置想低价转卖/交换的人；想省钱淘二手的人），以宿舍生活、课程教材、数码设备、日用品等高频物品为主。

### 2. 用户的主要痛点是什么？
答：
- 信息分散在朋友圈/群聊：难找、难搜、易错过
- 分类与质量信息不统一：同类物品混在一起，判断成本高
- 沟通效率低：反复问“还在吗/价格/自提地点/成色”
- 校内同城交易不便利：缺少统一入口与展示机制

### 3. 我们希望为用户提供什么帮助？
答：
- 提供统一发布入口与规范化字段（品类、价格/免费、成色、校内自提地点、联系方式）
- 提供搜索与分类浏览，快速找到需要的物品
- 提供“最新/热门”列表，降低发现成本
- 提供便捷的联系卖家方式（站内联系入口或联系方式展示）

## 四、页面目标

### 1. 这个主页最重要的作用是什么？
请从以下角度思考：
- 展示信息
- 引导使用
- 导航资源
- 介绍服务
- 联系或招募

答：展示与导航为主——让用户一进来就能“搜得到、看得全、发得快、联系上”。

### 2. 用户访问主页后，希望完成什么动作？
答：
- 通过搜索框输入关键词，进入结果列表/详情
- 点击分类（书籍/数码/生活用品/美妆）快速浏览
- 查看“最新发布/热门推荐”的物品卡片并进入详情
- 点击“发布”按钮，进入发布页完成快速发布
- 在详情页点击“联系卖家”并完成沟通发起

## 五、页面结构

请列出首页的主要结构：

- 顶部导航：Logo/项目名 + 搜索框（支持关键词搜索） + 分类入口（可为图标或标签）
- 主视觉区（Hero）：轮播推荐位（热门闲置 / 最新发布 / 校园自提提示）
- 功能介绍区：分类导航（书籍、数码、生活用品、美妆） + 发布入口按钮（固定或显眼）
- 其他内容区：最新发布列表（时间排序） + 热门推荐列表（浏览/收藏/点赞数等规则可先用模拟数据）
- 联系 / 底部区：使用须知（仅校内自提、注意安全）+ 反馈入口/联系我们（可先用表单或邮箱/群二维码占位）

## 六、功能要求

### 必做功能
请列出 2~3 个必须完成的内容。

- 必做功能 1：搜索与分类浏览（关键词搜索 + 分类筛选进入列表）
- 必做功能 2：物品列表与详情展示（图片/标题/价格或免费/成色/自提地点/发布时间）
- 必做功能 3：发布闲置入口与发布表单（选择分类、填写信息、提交后出现在“最新发布”）

### 可选功能
如时间允许可增加：

- 可选功能 1：个人中心（我的发布/编辑/下架）
- 可选功能 2：收藏/点赞与热门规则（提升“热门推荐”可信度）

## 七、交互要求

请至少设计 2 个可点击或可交互的地方，例如：

- 按钮点击跳转
- 导航定位到某一模块
- 展开 / 收起说明
- 切换展示内容

答：
- 交互 1：搜索框输入关键词后点击“搜索/回车”进入结果页（或在本页刷新列表）
- 交互 2：点击分类导航标签（书籍/数码/生活用品/美妆）切换列表内容
- 交互 3：点击“发布闲置”按钮跳转发布页；发布成功后返回首页并更新“最新发布”
- 交互 4：点击物品卡片进入详情页；点击“联系卖家”触发联系动作（展示联系方式/复制/跳转站内联系）

## 八、页面风格

请描述你们希望页面呈现的风格：

- 简洁 / 活泼 / 科技 / 学术 / 校园 / 专业 / 其他
- 主色调：建议偏校园清爽（如蓝/绿系，具体由设计稿确定）
- 关键词：校园感、清爽、信息卡片化、易检索、低干扰

答：整体风格“校园 + 简洁”，突出搜索与信息卡片，减少花哨装饰，保证一屏内能看到分类与最新内容。

## 九、验收标准

请写出你们小组的最低完成标准：

1. 页面可以正常打开
2. 首页结构完整
3. 至少有 4 个内容模块（搜索/轮播、分类导航、最新发布、热门推荐）
4. 至少有 3 个交互（搜索、分类切换、发布入口/物品卡片点击）
5. 页面文案基本通顺
6. 页面布局没有明显错乱

## 十、备注

还有哪些想法需要保留？

答：
- 项目边界：不做复杂支付；不做跨校；不做物流对接，仅支持校内自提/当面交换。
- 安全提示：在页面明显位置加入“校内公共区域交易、注意安全、防诈骗”等提示。
- 数据方案：演示版可先用模拟数据；如要上线再接入后端与账号体系。
